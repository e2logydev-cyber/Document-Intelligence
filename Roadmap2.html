End-to-End Pipeline
9-Step Data Flow — Blazor + Foundry Implementation

The complete proposal automation pipeline from RFP upload to final delivery, built entirely on .NET and Microsoft Foundry.

1
Ingestion · Blazor Server Frontend
Bid manager uploads RFP via Blazor web interface
The Blazor Server app provides a rich drag-and-drop upload interface with real-time progress indicators. Files are validated (PDF/DOCX only, max 50MB) and immediately streamed to Azure Blob Storage using the Azure.Storage.Blobs SDK with managed identity authentication.
Blazor Server + SignalR Azure AD Authentication Azure Blob SDK (Managed Identity) Real-time upload progress
→ File stored in Blob, metadata written to MS SQL, Foundry Agent session initialized
2
Orchestration · Foundry Agent Service
Foundry Agent creates durable session and initializes thread
The Blazor backend calls the Foundry Agent Service HTTP endpoint to create a persistent session. This establishes a thread that maintains state across all 9 steps. The agent receives the blob URL and proposal ID, storing both in its durable memory.
Foundry Component — Agent Session
Durable thread with persistent memory
Unlike Power Automate's stateless flow runs, the Foundry Agent maintains context across the entire proposal lifecycle. The thread survives restarts, retries, and can be queried later for audit and Q&A.
Session.Create() Thread Memory blobUrl stored proposalId stored
→ Agent thread active, ready to execute tool calls
3
Document Parsing · Azure Document Intelligence via MCP
Agent calls Document Intelligence MCP tool to parse RFP
The Foundry Agent invokes the Document Intelligence MCP tool, passing the blob URL. Foundry handles the async polling internally — the agent receives a structured extraction result containing client name, scope, requirements, budget, deadline, and compliance mandates.
Foundry MCP Approach
Agent calls: AnalyseDocument(blobUrl) → Foundry manages async polling → receives structured JSON extraction
MCP: Document Intelligence Async handled by Foundry Structured extraction → Agent memory
→ RFP structured data in agent thread, ready for search
4
Hybrid Search · MS SQL Vector + Azure AI Search
Parallel retrieval from MS SQL vector store and AI Search indexes
Using the extracted requirements as query vectors, the system runs parallel searches: (1) MS SQL vector similarity search for past projects and team profiles using native cosine similarity, and (2) Azure AI Search for reference documents and pricing templates. Both results are merged and ranked.
Hybrid Retrieval — MS SQL + AI Search
Native vector operations in SQL + Semantic search
MS SQL's new vector(1536) type stores embeddings natively. The system uses EF Core with the SqlServer.VectorSearch extension to perform cosine similarity directly in the database — no HTTP calls, no external services for structured data retrieval.
vector(1536) columns CosineSimilarity() EF Core VectorSearch Hybrid ranking
→ Top results from 4 sources assembled for GPT context
5
Context Assembly · Foundry IQ
Foundry IQ ranks and assembles optimal context package
Foundry IQ applies iterative reflection across the hybrid search results — evaluating whether the retrieved context is sufficient and re-querying if necessary. The final context package includes top 5 past projects, top 8 team members, relevant clauses, and pricing models — all with provenance tracking.
Iterative reflection Cross-source reranking Confidence thresholds Auto-citation generation
→ Optimized context package ready for generation
6
AI Generation · Foundry Model Catalogue
GPT-4o generates 5 proposal sections with structured output
The agent invokes GPT-4o through the Foundry Model Catalogue with structured output mode enabled. The model receives the assembled context and returns exactly 5 sections in validated JSON schema — each with content and citations array linking back to source documents.
Section 1
Executive Summary
Section 2
Technical Approach
Section 3
Team Profiles
Section 4
Pricing Narrative
Section 5
Project Timeline
→ Validated JSON stored in agent thread, ready for persistence
7
Persistence · MS SQL via EF Core
Agent writes sections to MS SQL, Blazor SignalR pushes updates
The Foundry Agent calls a custom MCP tool that executes EF Core operations — creating ProposalSection records in MS SQL with full transaction support. Blazor's SignalR hub detects the database changes via SQL Server query notifications and pushes real-time updates to all connected clients.
EF Core + MS SQL SignalR real-time sync SQL Server Query Notifications Agent thread retains full history
→ Draft visible in Blazor workbench within seconds of generation
8
Review Workflow · Blazor + Email Integration
Role-based review in Blazor with email notifications
Reviewers access tailored views in the Blazor app — Tech Leads see technical sections, Commercial Directors see pricing. The Foundry Agent sends email notifications via the Email MCP tool with deep links back to the specific proposal section in Blazor. Review comments are stored in MS SQL and trigger agent re-processing if changes are requested.
Blazor role-based views MCP: Email notifications Deep link routing Agent handles revision loops
→ Full review cycle without leaving the Blazor ecosystem
9
Delivery · Automated Export + Full Trace
Final proposal generated, delivered, and fully traced
On approval, the agent orchestrates final delivery: (1) MCP call to generate PDF/DOCX using DocumentFormat.OpenXml, (2) Blob storage upload with SAS token, (3) MS SQL status update, (4) Email notification with download link. The entire execution trace is preserved in Foundry Control Plane for audit.
Foundry Control Plane — Full Observability
Every action traced, evaluated, and auditable
Unlike Power Platform's limited logging, Foundry Control Plane captures every MCP tool call, model invocation, retrieval query, and output. Built-in evaluators score groundedness, coherence, and safety. Failed proposals can be replayed step-by-step.
Full execution trace Built-in evaluators Token usage tracking Replay capability
→ 40%+ reduction in time-to-submit vs. manual baseline
