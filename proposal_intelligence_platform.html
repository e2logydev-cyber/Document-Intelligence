05
Phase 1

Upload & AI Processing

The Uploader submits the RFP via the Blazor UI. The .NET API routes the file to Azure Blob and triggers Microsoft Foundry. The agent calls Azure Document Intelligence to extract requirements, then Azure AI Search to retrieve relevant past proposals, and finally ChatGPT to draft each section in the company's voice.

📤
Blazor Upload UIUploader selects RFP file; progress tracked in real time via SignalR push.
📄
Document IntelligenceAzure extracts structured requirements, dates, and sections from the PDF automatically.
🔍
AI Search (RAG)Foundry queries the index for semantically similar past proposals to inject as context.
💬
ChatGPT DraftingLLM drafts each proposal section in the company's voice with full RAG context.
06
Phase 2

Review & Approval

Once the draft is ready, SignalR pushes a real-time notification to the assigned Proposal Manager. The manager reviews, annotates, and routes sections to Subject Matter Experts. Each review decision is recorded in MS SQL. Approvals flow through a structured chain — no more email threads.

🔔
SignalR NotificationReal-time push alert fires the moment the AI draft is ready for review.
✏️
Inline EditorProposal Manager edits sections, adds comments, and assigns reviewers directly in Blazor.
👥
SME AssignmentSections routed to Subject Matter Experts by role. Each section has a clear owner and deadline.
✅
Approval ChainStructured approve / reject / request-changes workflow persisted to SQL with full audit trail.
07
Phase 3

Generate, Share & Roles

After final approval, Microsoft Foundry orchestrates document generation — assembling the complete proposal as a formatted file in Azure Blob. It then publishes directly to SharePoint, notifies stakeholders, and the RBAC layer enforces who can view, edit, or export each proposal going forward.

📝
Document GenerationFoundry assembles final proposal DOCX/PDF from approved sections. Stored in Azure Blob.
📁
SharePoint PublishFinal document pushed to SharePoint library automatically. Accessible to permitted stakeholders.
🔐
Role-Based AccessRBAC controls who can view, download, or re-open the proposal for revision.
📊
Audit & AnalyticsEvery action logged. Dashboards show cycle times, win rates, and contributor activity.
