Pipeline Overview
1
Ingestion · Blazor / Power Apps
Bid manager uploads client RFP
The proposal process begins when a bid manager uploads the client RFP — as a PDF or DOCX — directly from the Blazor workbench on desktop or via the Power Apps mobile companion app in the field.
PDF / DOCX upload Blazor workbench Power Apps mobile Drag-and-drop or camera capture
→ Triggers the full AI pipeline automatically
2
Orchestration · Power Automate
Power Automate flow triggered
The RFP Intake Pipeline flow activates immediately upon file upload — uploading the file to Azure Blob Storage and creating a Dataverse Proposal record to track status throughout the lifecycle.
Upload to Azure Blob Storage Create Dataverse Proposal record Status = RFP Received Initialize pipeline variables
→ Dataverse becomes the single record-of-truth
3
Document Parsing · Azure Document Intelligence
Azure Document Intelligence parses the RFP
The prebuilt document model analyses the uploaded RFP, extracting all structured data points needed for AI search and proposal generation. Results are written to the RFPDocument Dataverse table.
Client name & project scope Technical requirements Evaluation criteria Budget range Submission deadline Compliance mandates
→ Structured JSON written to RFPDocument table
4
Semantic Search · Azure AI Search — 4 Indexes in Parallel
All four knowledge indexes queried simultaneously
Using extracted requirements as the semantic query vector, Azure AI Search runs parallel queries across all four knowledge indexes using Power Automate's "Run in Parallel" scope — maximising retrieval speed and relevance.
Past Projects
Case studies, SOWs, post-mortems · similarity scores, tech stack, reusable content
Team Profiles
Skills matrix, availability, certifications · matched against RFP tech requirements
Ref Documents
Compliance checklists, legal T&Cs, boilerplate · template population & compliance checks
Pricing Templates
Rate cards, effort models, margin guidelines · commercial section drafting
→ Semantic vector search with confidence thresholds on all 4 indexes
5
Results Assembly · AI Search
Top results assembled from all knowledge sources
Search results from all four indexes are ranked and assembled into a structured context package, ready to be passed to GPT-4o for proposal generation.
Top 5 similar past projects with similarity scores Top 8 available team members Relevant reference document sections Applicable pricing templates
→ Full context package assembled for GPT-4o
6
AI Generation · Azure OpenAI GPT-4o
GPT-4o generates structured draft sections with citations
GPT-4o receives the assembled context — RFP extract, top 3 past projects, recommended team, applicable pricing templates, and relevant reference sections — and generates five complete proposal sections in structured JSON, each with inline citations linked back to source documents.
Section 1
Executive Summary
Section 2
Technical Approach
Section 3
Team Profiles
Section 4
Pricing Narrative
Section 5
Project Timeline
→ Each section includes citations[] linked to knowledge source names
7
Storage & Display · Dataverse + Blazor
Draft saved to Dataverse · displayed in Blazor workbench
All five AI-generated ProposalSection records are written to Dataverse. The Blazor workbench immediately renders the sections side-by-side with inline AI citations — each citation is a live link back to the source past project, team member, reference document, or pricing template.
ProposalSection records written to Dataverse Inline citations linked to knowledge sources Section editor open for human refinement Version history tracked automatically
→ Bid manager reviews AI draft in under 3 minutes from upload
8
Review & Approval · Microsoft Teams Adaptive Cards
Tech Lead & Delivery Manager review via Teams
Each approver receives a personalised Teams Adaptive Card showing only the sections relevant to their role. Tech Leads see the Technical Approach; Delivery Managers see the Team & Timeline; Commercial Directors see the Pricing Narrative — with one-click actions.
Tailored Adaptive Card per reviewer role Approve / Request Changes / Escalate actions Inline comments from Teams Sequential approval routing
→ Full review cycle without leaving Microsoft Teams
9
Delivery · Power Automate + Power BI
On approval — proposal packaged and delivered
The Proposal Export & Deliver flow assembles the final proposal, packages it as both PDF and DOCX, uploads to SharePoint, updates the CRM opportunity stage, and notifies the Bid Manager. Power BI dashboards refresh pipeline metrics automatically.
PDF + DOCX generated and uploaded to SharePoint CRM opportunity stage updated Bid Manager notified via Teams & email Power BI pipeline metrics refreshed
→ 40%+ reduction in time-to-submit vs manual baseline
